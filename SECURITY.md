# Security Policy

We take the security of MailCue seriously. If you believe you have found a security vulnerability, please report it to us responsibly as described below.

## Reporting a Vulnerability

Please do not open a public GitHub issue for security-related bugs. Instead, email reports to **[hello@olib.ai](mailto:hello@olib.ai)**.

Please include:
*   A detailed description of the vulnerability.
*   Steps to reproduce or a proof of concept (PoC).
*   Any potential impact or scenarios where this could be exploited.

We will acknowledge receipt of your report within 48 hours and work with you to resolve the issue promptly.

## Supported Versions

Only the latest release of MailCue is actively supported with security updates. We recommend always running the latest Docker image (`ghcr.io/olib-ai/mailcue:latest`).
